Mutability

(Variables are immutable by default)


    let i = 5;

    i = 10;     // NOPE!
error: re-assignment of immutable variable `i` 

    let mut i = 5;  // explicitly mark it as `mut`

    i = 10;   // Okay now!

Ownership

(Bindings have "ownership" on what they're bound to)


    struct Book { name: String }

    fn take(_: Book) {}

    fn main() {
        let book = Book { name: String::from("Robin Hood") };
        take(book);     // Book taken!

        println!("{}", book.name);    // Ouch! Book gone!
    }
error: use of moved value: `book`

Not everything's moved though!


    #[derive(Clone, Copy)]    // mark it as copy
    struct Foo(i32);

    fn take<T>(_: T) {}       // take anything

    fn main() {
        let x = 1;
        take(x);
        println!("{}", x);    // works
        let f = Foo(30);
        let y = f;
        println!("{}", f.0);  // yep, still works!
    }

Types that can be shallow copied aren't moved.

The rest will be moved!


    let vector = vec![1, 2, 3];   // vector
    let stolen = vector;
    println!("{:?}", vector);     // error: moved!

    let s = "Hello, world!";
    take(s);
    println!("{}", s);        // strangely, this works!

    let owned = String::from("Hello, world!");
    let stolen = owned;
    println!("{}", owned);    // but, this doesn't!

But, that's because there are two versions of values.

"Great! How do you do stuff, then?"

Well, you use it and give it back!

(or, get another copy)


    #[derive(Clone)]
    struct Book { name: String }

    fn take_and_give(book: Book) -> Book { book }

    fn take(book: Book) {}

    fn main() {
        let book = Book { name: String::from("Robin Hood") };
        let book = take_and_give(book);
        take(book.clone());     // clone (get another copy)
        println!("{}", book.name);    // Yay!
    }

Or, (instead of stealing), you could just "borrow" it.

Borrowing

(fancy name for taking references)


    struct Book { name: String }

    fn take(book: &Book) {}   // note the function signature

    fn change(book: &Book) {
        *book = Book { name: String::from("Harry Potter") };
    }

    fn main() {
        let book = Book { name: String::from("Robin Hood") };
        take(&book);    // note the reference
        change(&book);  // nope!
    }
error: cannot assign to immutable borrowed content `*book`

"Okay, what if you wanna change stuff?"

You mark it as mutable!


    fn read(book: &Book) {
        println!("{}", book.0);
    }

    fn exchange(book: &mut Book) {    // mutable reference
        *book = Book { name: String::from("Harry Potter") };
    }

    let mut book = Book { name: String::from("Robin Hood") };
    read(&book);    // "Robin Hood"
    exchange(&mut book);    // we're allowing the change
    read(&book);    // "Harry Potter"

(... might as well get a new book)

Many readers, one writer.


    let mut a = vec![1, 2, 3];
    {
        let b = &a;   // borrowed
        a.push(4);    // doesn't work
    }   // borrow ends here
error: cannot borrow `a` as mutable because it is also borrowed as immutable 

    let mut a = vec![1, 2, 3];
    {
        let b = &a;
        println!("{}", a[0])  // immutable reference
    }
    a.push(4);    // we can mutate now

Many readers, one writer.


    let mut a = vec![1, 2, 3];
    {
        let b = &mut a;   // mutably borrowed
        let c = &mut a;   // cannot borrow again!
    }   // borrow ends here
error: cannot borrow `a` as mutable more than once at a time 

    let mut a = vec![1, 2, 3];
    {
        let b = &mut a;
        b.push(4);    // works (mutable reference)
    }
    let c = &mut a;   // works now

    println!("{}", a[0]);   // doesn't work (mutably borrowed)
