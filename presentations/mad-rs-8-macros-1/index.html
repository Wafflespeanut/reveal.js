Declarative macros

Pros:

  • In the first phases of compilation.
  • Variadic function-like.
  • match-like.

Cons:

  • Code writing Rust code. Hard to read/maintain.
  • Need to export, import and bring it into scope.
  • Sometimes hard to debug.

Example: try!


    macro_rules! try {      // NOTE: Superseded by `Try` trait.
        ($expr:expr) => {
            match $expr {
                Ok(v) => v,
                Err(e) => return Err(e.into()),
            }
        };
    }
            

    fn main() -> Result<(), YourError> {
        let mut buf = [0; 10];
        let fd = try!(File::open("/dev/urandom"));
        try!(fd.read_exact(&mut buf));
        Ok(())
    }

Example: assert_eq!


    macro_rules! assert_eq {    // NOTE: different from stdlib
        ($l:expr, $r:expr) => {
            if $l != $r {
                panic!("assertion failed: `(left == right)`
   left: `{:?}`,
  right: `{:?}`", $l, $r);
            }
        };
    }
            

    fn main() {
        assert_eq!("hello", "hello");
    }

Pattern types

  • stmt: a statement
  • pat: a pattern
  • expr: an expression
  • ty: a type
  • ident: an identifier
  • tt: a single token tree
  • item: an item (function, struct, module, etc.)
  • block: a block (of stmts and/or exprs in braces)
  • path: a path (e.g. foo, crate::bar::baz)
  • meta: stuff inside #[...] and #![...] attributes

Repetition example: vec![]


    macro_rules! vec {    // NOTE: different from stdlib
        ($e:expr; $n:expr) => {
            let mut v = Vec::with_capacity($n);
            v.extend($n, iter::repeat($e).take($n));
            v
        };
        ($($x:expr),*) => (
            <[_]>::into_vec(Box::new([$($x),*]))
        );
    }
            

    fn main() {   // capacity is set to exactly what's required.
        let v1 = vec![0; 5];
        let v2 = vec![1, 2, 3, 4, 5];
    }
