Functions: 3 kinds

  • Fn: immutable capture; doesn't mutate context
    (can be called indefinitely)
  • FnMut: mutates context
    (can be called indefinitely)
  • FnOnce: captures (moves) stuff from context
    (can be called only once)

Fn


    // prints length of a string using the provided counter.
    fn print_string_len<F>(s: &str, counter: F)
        where F: Fn(&str) -> usize
    {
        println!("Length of {}: {}", s, counter(s));
    }
            

    let string = "❤️";
    let bytes_counter = |s: &str| s.len();
    let char_counter = |s: &str| s.chars().count();

    print_string_len(string, byte_counter);
    print_string_len(string, char_counter);

FnMut


    let mut chars: usize = 0;
    let mut count_and_print = |s: &str| {
        chars += s.chars().count();
        print!("{}", s);
    };

    count_and_print("Hello");     // assert_eq!(chars, 5);
    count_and_print(" world!");   // assert_eq!(chars, 12);
  • Fn is a subtrait (implements FnMut)
  • Used by Iterator methods

FnOnce


    let (tx, rx) = ::std::sync::mpsc::channel();
    let expensive_func = move || {
        for i in 0..5 {
            let time = ::std::time::Duration::from_millis(1000);
            ::std::thread::sleep(time);
            let _ = tx.send(i);   // send every second
        }
    };

    ::std::thread::spawn(expensive_func);
    while let Ok(i) = rx.recv() {
        println!("Got {}", i);
    }
  • Fn and FnMut are subtraits (implement FnOnce)
  • Used by Future methods

Real world example: HTTP client

(Beware: complexity)

  • Hyper: High-performant HTTP lib in Rust.
  • rustls: Modern TLS lib written in Rust.
  • hyper-rustls: Rustls integration for hyper (HTTP lib).
  • Alternative for native TLS.
  • Very useful for static builds.

HTTP client: Types


    use hyper::client::{Client, HttpConnector};
    use hyper_rustls::HttpsConnector;

    pub type YourClient = Client<HttpsConnector<HttpConnector>>;
            

    pub type ClientRequest
        = Box<Fn(&YourClient) -> YourFuture<()> + Send + 'static>;

HTTP client: Setup


    use futures::{Future, Stream, future};
    use futures::sync::mpsc::{self, Sender};
    use tokio_core::reactor::Core;
            

    pub fn prepare() -> Sender<ClientRequest> {
        let (sender, receiver) = mpsc::channel(0);
        thread::spawn(move || {
            let mut core = Core::new().unwrap();
            let connector = HttpsConnector::new(4);
            let client = Client::builder().build(connector);
            let listen = receiver.for_each(|call: ClientRequest| {
                call(&client)
            });
            core.run(listen).unwrap();
        });
        sender
    }

    let req = Request::get("https://www.rust-lang.org")

    let client = ... ;
    let future = client.request(req).and_then(|resp| {
        let code = response.status();
        response.into_body().fold(vec![], |mut body, ref chunk| {
            body.extend_from_slice(chunk);
            future::ok(body)
        }).and_then(|bytes| {
            println!("code: {}, response: {}", code,
                     String::from_utf8_lossy(&bytes));
        })
    });
